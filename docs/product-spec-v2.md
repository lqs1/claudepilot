# ClaudePilot 产品文档 (v2)

> **文档定位**：以"用户价值"为主线，而非"CLI flag 翻译"。每一项功能必须同时命中
> 三个条件才进入路线图：**用户有真实需求 × CLI 当前做不到/做不好 × Web 能给出独特价值**。
> 与 [product-spec-v1.md](./product-spec-v1.md)（CLI 全景直译版）并存，后者作为参考。

## 1. 产品定位

ClaudePilot 是 Claude Code CLI 的 **Web 价值放大器**。

- **CLI 是引擎**，ClaudePilot **不替换引擎、不重写模型调用**，只做编排、可视化和增强。
- **不是"Web 版 CLI"**。用户切到 Web 版的理由永远是：做 CLI **做不到 / 做不好**的事，
  而不是把 CLI 的每个 flag 复刻成按钮。后者维护负担无限、且永远是落后版本。
- **一句话价值主张**：一站式管理多个项目与会话，可视化每次变更，把重复配置沉淀成预设，
  并行驱动多任务——这些都是单进程 CLI 的硬伤，正是 Web 的主场。

### 与 CLI 的分工边界

| 方面                | CLI 的角色                      | ClaudePilot 的角色                         |
| ------------------- | ------------------------------- | ------------------------------------------ |
| 模型调用 / 工具执行 | **唯一执行者**（子进程）        | 只做编排和转发，绝不自己调模型/跑工具      |
| 会话上下文记忆      | `~/.claude` jsonl（**真相源**） | 读/展示/编辑，不另存一份会话内容           |
| 配置参数            | 启动 flag                       | 图形化 + **预设/模板**（CLI 做不到的沉淀） |
| 多会话              | 单进程弱                        | **并行多 session** 是杀手级能力            |
| 变更可视化          | 文本输出                        | diff / 文件树 / 调用链 / 用量曲线          |

## 2. 核心原则（驱动所有取舍）

1. **做短板，不做对等**：每个功能问"CLI 为什么做不了/做得差"。答不上来的，不做。
2. **CLI 是移动靶**：2.1.x 迭代极快，flag/event/斜杠命令随时变。
   **禁止硬编码 flag 枚举表**；前端能力选项必须运行时探测（见 §7）。
3. **真相单一**：会话历史 = CLI 的 jsonl（已完成同源）。配置的真相源也是 CLI/文件，
   UI 只做缓存和写入，不存第二份权威数据。
4. **降级而非崩溃**：CLI 新增未知 event / 未知 flag / 未知斜杠命令 → 系统继续工作，
   只是把"未知项"作为原始文本展示或隐藏，永远不打断用户。
5. **能省则省**：能用原生文件/CLI 命令完成的（如编辑 agents、MCP 配置），
   不在 Web 里重造编辑器，只提供"打开文件"的快捷入口。

## 3. 路线图：聚焦做 / 明确不做

### ✅ MVP（已完成）

| 能力                                                  | 状态 |
| ----------------------------------------------------- | ---- |
| 项目 + 多会话管理（本地路径）                         | ✅   |
| 聊天：流式渲染、用户消息、工具卡片、Markdown          | ✅   |
| **会话历史与 CLI jsonl 同源**（读 / 配对删除 / 续聊） | ✅   |
| WebSocket 实时推送 + 状态指示                         | ✅   |
| 终端（xterm.js + PTY）、文件浏览器 + Monaco           | ✅   |
| AskUserQuestion / Permission 弹窗、Plan Mode          | ✅   |
| 中英双语（UI + 系统提示）                             | ✅   |

### 3.1 P0 — 真正的差异化价值（立即聚焦）

这些是**CLI 给不了、用户每天都想要**的能力：硬重点。

| 能力                       | CLI 的短板                       | ClaudePilot 的独特价值                                                                  |
| -------------------------- | -------------------------------- | --------------------------------------------------------------------------------------- |
| **会话/项目持久化与检索**  | 跨 session 切换弱、难检索        | 单一入口管理所有项目/会话，历史同源可搜索                                               |
| **历史检索 + 删除 + 续聊** | jsonl 没有前端、删会话只能删文件 | 全文检索、按轮次安全删除、一键续聊（✅ 已完成基础）                                     |
| **变更可视化**             | 只有文本 stdout                  | diff 视图、文件变更树、工具调用链时间线、token/用量/成本曲线                            |
| **配置预设（Preset）**⭐   | 每次都要背 flag                  | 把 `(model + effort + permission + system-prompt + tools + mcp)` 存成命名预设，一键应用 |
| **批量/并行会话**⭐        | 单进程串行                       | 对同一项目开 N 个并行 session 同时跑不同子任务                                          |

> ⭐ 标记的两项是最高 ROI：Preset 把 CLI 的"配置复杂"变成"一次性沉淀"；
> 并行会话是 CLI 架构上做不到的能力。这两件事才让 Web 版有不可替代性。

### 3.2 P1 — 体验增强（按需）

| 能力                           | 说明                                    | 价值                         |
| ------------------------------ | --------------------------------------- | ---------------------------- |
| 用量看板                       | 聚合 token / 缓存命中 / 成本            | CLI 散落各处，聚合后才看得清 |
| 提示建议（prompt-suggestions） | 复用 CLI 的 `--prompt-suggestions` 事件 | 数据现成，做成快捷按钮即可   |
| 会话命名 / 标签                | 复用 CLI `--name`                       | 多会话管理的必备导航         |
| 重试 / 分叉会话                | 复用 `--fork-session`                   | 从某轮分叉探索不同方向       |
| 预算控制                       | 复用 `--max-budget-usd`                 | 防失控花费                   |

### 3.3 P2 — 次要 / 可选（确认有人用再做）

- MCP 服务器的"启用/禁用"开关（**不做配置编辑器**，复杂配置让用户直接编辑文件，UI 只提供"打开配置目录"）。
- `doctor` / 项目状态的诊断视图（只读展示 CLI 输出，不自己判断）。
- 后台 agent 状态面板（只读，对应 `claude agents`/`--bg`）。

### 3.4 ❌ 明确不做（避免无底洞）

| 不做的事                               | 理由                                                                          |
| -------------------------------------- | ----------------------------------------------------------------------------- |
| 把 30+ 个 flag 全做成控件              | 95% 用户只用 4-5 个（model/effort/permission/tools）；其余进预设              |
| 复刻每个斜杠命令为按钮                 | 斜杠命令多、语义随上下文变、迭代快；**直接让用户在输入框用 `/`** 更省更不脱节 |
| 桌面宠物 / 装饰性 Remote Control       | 与核心价值无关，纯维护负担                                                    |
| 自己实现 skills/plugins/agents 管理 UI | 这些有官方管理方式，复刻必然落后；提供"打开目录"入口足矣                      |
| vim/fuzzy 等 TUI 专属交互              | 搬到 Web 反而更难用，违背用户习惯                                             |

```

│  ClaudePilot                              [语言] [设置]      │
├────────────┬─────────────────────────────────────────────────┤
│ 项目       │  [Chat]  [Files]  [Terminal]  [Changes]  ...    │
│ ▸ proj-1   │─────────────────────────────────────────────────│
│ ▸ proj-2   │                                                 │
│            │   主内容区（按标签切换）                          │

│ ▸ sess-1   │   · Chat: 对话 + 工具卡片 + 状态                 │

│            │                                                 │

```

新增 **Changes** 标签和 **Preset 选择器**，是 P0 的落地位置。其余标签沿用现有设计。

## 5. 关键设计决策

1. **一个 Session = 一个 CLI 子进程**，以 `--session-id`(UUID) 绑定。
2. **会话历史真相 = `~/.claude/projects/<encoded-dir>/<sid>.jsonl`**，ClaudePilot 只读+编辑，
   不另存 SQLite 副本（已完成）。SQLite 仅存项目/会话**元数据**。
3. **配置真相源 = CLI 自身的文件**（settings.json / mcp / agents）。
   UI 缓存展示，写入直接改文件。不发明第二套配置格式。
4. **Permission / Question 代理**：CLI 请求 → 后端转发 → 前端弹窗 → 回传 stdin。
5. **删除安全**：配对删除一个轮次（user + 其 reply + 工具调用），保持 parentUuid 链完整，
   确保 CLI 仍可 `--resume`（已实现并验证）。
6. **历史是 append-only 的真相**：任何改写都是危险的，必须有确认；引擎运行时禁止改写（返回 409）。

## 6. 与 v1 文档的差异说明

[product-spec-v1.md](./product-spec-v1.md)（原 `product-spec.md`）是一份"CLI 功能全景 → 控件映射"清单，
把 `--model/--effort/--mcp-config` 等逐个做成控件。本 v2 文档**有意识收敛**：

- 不再逐 flag 做控件，改为**预设**承载大多数配置（v1 把这些塞进 Settings 标签，维护发散）。
- 把"复刻斜杠命令"降级为"在输入框直接用 `/`"（v1 计划逐个做按钮）。
- 砍掉桌面宠物、Remote Control 展示等与核心价值无关项。
- 新增 P0：**变更可视化、预设系统、并行会话**——这些 v1 完全没有，却是 Web 版的立身之本。
- 新增 §7 工程原则：CLI 版本解耦，根治 v1"文档一写就过时"的问题。

## 7. 工程原则：与 CLI 版本解耦 ⭐

CLI 是移动靶。当前 2.1.186 的 `--effort` 值是 `low/medium/high/xhigh/max`、有 `--max-budget-usd`、
`--name`、`--worktree`、`--brief` 等——这些每版都可能变。**任何硬编码的 flag/value 枚举都是技术债。**

**规则：**

1. **运行时探测**：后端启动时解析 `claude --help`，缓存"可用 flag + 合法 value"，前端选项据此动态渲染。
   - model 列表 → 探测或允许自由输入（用 alias），不硬编码模型 id。
   - effort/permission-mode 的可选值 → 探测，不写死。
2. **未知即透传**：CLI 新增的 flag/事件/斜杠命令，后端不认识时**原样透传或展示**，不打断、不报错。
3. **事件解析器宽容**：`StreamEventParser` 对未知 event type 走 raw_output 兜底（已实现）。
4. **flag 构造器零硬编码**：`ClaudeCommandBuilder` 只组装用户选择的 flag，不做"已有/废弃"判断。

> 有了这一层，ClaudePilot 才可能跟上 CLI 的迭代节奏，而不是每版都重写。

## 8. 数据存储

| 数据              | 位置                                       | 真相源              |
| ----------------- | ------------------------------------------ | ------------------- |
| 会话历史（消息）  | `~/.claude/projects/<encoded>/<sid>.jsonl` | CLI ✅              |
| 项目 / 会话元数据 | `~/.claudepilot/claudepilot.db`（SQLite）  | ClaudePilot         |
| 配置预设          | `~/.claudepilot/claudepilot.db` 或文件     | ClaudePilot（增值） |
| 配置本身          | CLI 各自的 settings/agents/mcp 文件        | CLI ✅              |

## 9. 非目标 / 拒绝清单

明确写出来，防止后续 scope 蔓延：

- ❌ 不自建模型客户端、不绕过 CLI 直接调 API。
- ❌ 不存储会话内容的第二份副本（只用 jsonl）。
- ❌ 不做 skills/plugins/agents 的**编辑** UI（只做入口/只读状态）。
- ❌ 不追求与 CLI **功能完全对等**——对等等于没有价值。
- ❌ 不逐个 flag / 斜杠命令做控件和按钮。

---

## 10. 验收标准（每个功能进 P0 的硬门槛）

1. 能说清 **CLI 为何做不到/做不好这一点**。
2. 实现后**可被版本解耦**（§7），不会因 CLI 升级而立即失效。
3. **降级安全**：CLI 缺该能力时，功能隐藏而非报错。
4. 不引入第二份**真相副本**（除非是纯增值的本地数据，如预设）。
